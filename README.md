# OneView 2.0 — Multi-Tenant Health Screening Dashboard

The complete Phase-1 admin platform for Zeikon Global's AI screening tools.
One clean codebase, one SQL file, plain-English errors. Built to run alongside
the old dashboard on the SAME database — nothing to migrate, nothing at risk.

## What's inside (Phase 1 — complete)
- Login with 3 levels: Super admin (you) / Client owner / Client staff (view-only)
- Clients: create, edit with explicit 💾 Save buttons, suspend, delete
  (type-code-to-confirm), scan limits, validity dates
- AI Tools catalogue with per-tool scan app URL (the field that makes links real)
- Links: per-tool links AND 🧭 Landing page links (one link → tile per enabled
  tool, Aurae-style), with per-link form-field checkboxes, QR codes, expiry,
  scan quotas, pause/resume, delete
- Dashboard logins management per client: invite by email or set password,
  change role, remove — powered by a secure server route
- Full analytics: KPI cards (totals / this month / previous / today, doctors
  engaged, balance ring), rich filters, charts, sortable table, Excel export
- Errors appear in plain English with the fix written next to them

## Setup (new project — old dashboard stays untouched)
1. **Database**: Supabase → SQL Editor → delete all old snippets → paste
   `ONEVIEW2_SETUP.sql` (edit the email near the bottom to yours) → Run.
   Bottom of results must show your email as super_admin.
   (If you already ran V10_FINAL, this is the same rules — running again is safe.)
2. **GitHub**: create a NEW repo `oneview2` → upload this folder's contents
3. **Lovable**: create a NEW project → connect GitHub repo → connect the SAME
   Supabase project (never click "Enable Cloud") → add secret
   `ONEVIEW_SERVICE_ROLE_KEY` = Supabase secret key → Publish
4. **AI Tools tab** → Face Vitals → paste the face scan app's published URL
   into "Scan app URL" → Save
5. Test: Clients → Alkem → 🧭 Landing page link → create → open on phone

## The permanent rules
- GitHub = where code lives. Lovable = runs the code. Supabase = where data lives.
- Only ever run `ONEVIEW2_SETUP.sql`. Never any other SQL file, never Lovable's
  self-invented "fixes".
- Links are generated by the system; the Label box is just a nickname.
